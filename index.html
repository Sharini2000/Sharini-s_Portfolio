üìò Book Recommendation System

The Book Recommendation System project showcases my expertise as a Full Stack Developer, where I developed a web-based application designed to provide book recommendations, sentiment analysis, and topic-based feedback based on user queries. Leveraging Natural Language Processing (NLP) and a knowledge graph, the system analyzes book reviews and suggests relevant books by integrating a Flask and MySQL backend with a React frontend. Key technical features include book information and sentiment analysis, which extracts metadata (author, category, publisher) using a knowledge graph and analyzes review sentiment with TextBlob to determine overall book perception. The system employs Latent Dirichlet Allocation (LDA) to extract key discussion topics from book reviews, providing summarized insights into key book-related themes, and offers entity-based recommendations by identifying key entities (author, characters, themes) from reviews using SpaCy, suggesting books with overlapping entities and topics. WebSocket communication enables real-time responses using Flask-SocketIO, supporting both HTTP webhooks and WebSocket-based interactions. For testing, I utilized Postman for API testing, unit tests for recommendation algorithms and NLP, Socket.IO client for WebSocket testing, and manual UI testing for frontend validation. The deployment journey began with an AWS attempt using ECS (Docker) for the Flask backend, RDS for the MySQL database, and Amplify for the React frontend, though high costs and AWS free tier limitations posed challenges. Development tools included VS Code and Google Colabs, with GitHub for version control.

üå¶Ô∏è Weather Classifier (2024)

The "Weather Classifier Using Machine Learning Model" enhances weather prediction for agriculture, transportation, and disaster management. It addresses missing data via preprocessing, categorizing days and using filter/wrapper methods for feature interaction. EDA uncovers correlations through histograms, scatter plots, and box plots (e.g., temperature vs. dew point). Methodology includes concatenating 12 monthly CSV files, feature engineering with hourly data, season determination, and temperature difference calculations. Feature selection employs Mutual Information and a decision tree to identify key variables (season, weekday, temperature difference). Training compares SVM and RF using 3-fold cross-validation and GridSearchCV, with results showing RF outperforming SVM via stratified k-fold and hyperparameter tuning. Random Forest emerges as the top algorithm, improving meteorological forecasts by identifying critical factors like humidity and station pressure.
